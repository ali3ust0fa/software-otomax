Block Ids Which Contains

Acapkali terjadi kasus dimana id Jabber/XMPP Reseller dikuasai oleh pihak lain secara tidak sah dan digunakan untuk bertransaksi yang mengakibatkan kerugian kehilangan sejumlah saldo, dan setelah dianalisa id Jabber/XMPP tersebut digunakan login (NAMUN TIDAK TERBATAS) di pandion misalnya: id_jabber@jabber.com/pandion; atau id-nya: penembak_nakal misalnya: penembak_nakal@jabber.com/SD9939F9F9; atau domain-nya: jab.er misalnya: id_jabber@jab.er.

Guna melindungi Reseller dari kasus seperti diatas mulai versi 3.9.0 Anda bisa menggunakan fitur Block Ids Which Contains yang dapat memblokir id Jabber/XMPP yang mengandung kalimat pandion, penembak_nakal, jab.er atau kalimat lain sejenis, sehingga tidak dapat bertransaksi.

Berikut ini langkah - langkah menggunakan fitur diatas:

  1. Pastikan Anda mempunyai OtomaX, kalau belum klik disini.
  2. Pastikan telah menggunakan OtomaX versi 3.9.0 atau lebih tinggi, kalau belum update dulu klik disini. Jika masa berlaku habis bayar iuran dahulu klik disini.
  3. Pilih Modul IM Center -> klik kanan -> klik Setting -> klik tab Global Settings -> pada box: Block ids which contains misalnya isi dengan pandion dan kemudian klik tombol Save.



    Apabila ingin menambahkan kalimat yang ingin diblokir lebih dari satu, pisahkan dengan tanda koma, contoh: pandion,penembak_nakal,jab.er.

  4. Selesai.

Selanjutnya apabila Anda mempunyai pertanyaan seputar fitur diatas silahkan kirim ke alamat email support@otomax-software.com.
