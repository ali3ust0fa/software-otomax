IP Client (3.6.8)

Modul ini sudah tersedia di versi sebelumnya, yakni sejak versi 3.1.3 di semua edisi berbayar klik disini, berguna untuk melakukan host to host ke server lain via IP dengan metode yang didukung XML/Text, HTTP GET dan HTTP POST klik disini. Terkait tata cara penggunaan IP Client (3.6.8) selengkapnya klik disini

Di versi 3.6.8 IP Client mendapat pembaharuan berupa mendukung Sign versi OtomaX (belum mendukung Sign versi lain) klik disini dimana ini berguna mengamankan PIN dan Password ketika transaksi IP. Sign merupakan tanda unik yang digenerate otomatis oleh OtomaX dari parameter tertentu dengan rumus tertentu dan berubah - ubah mengikuti parameternya klik disini.

Penggunaan Sign versi OtomaX pada IP Client akan efektif jika Supplier Anda menggunakan Software OtomaX dan edisinya Enterprise atau Ultimate, serta menggunakan versi minimal 3.6.8. Oleh karena itu bila Supplier Anda belum memakai OtomaX arahkan untuk membeli OtomaX klik disini; dan bila OtomaX Anda atau Supplier Anda bukan edisi Enterprise atau Ultimate upgrade dahulu klik disini; dan bila OtomaX Anda belum versi 3.6.8 update dahulu klik disini.

Perlu Anda ketahui Sign tidak dapat Anda buat sesuai kebutuhan Anda (tidak custom), melainkan Sign dibuat built-in oleh OtomaX secara otomatis. Jadi misalnya Supplier Anda spesifikasi transaksi IP-nya mengharuskan penggunaan Sign yang bukan Sign versi OtomaX, maka Anda tidak dapat melakukan host to host ke dia via IP klik disini. Salah satu solusinya ialah arahkan Supplier Anda untuk membeli OtomaX klik disini.

Kemudian bila Anda punya pertanyaan atau permasalahan terkait transasi IP (Support Berbayar) dapat menghubungi Kami, adapun Support Gratis klik disini

Catatan 1: Pastikan Anda telah meng-klik kalimat klik disini dan membaca semua isinya agar lebih memahami transaksi IP, ingat kata ibu guru: "Rajin Pangkal Pandai".

Catatan 2: Mulai versi 3.7.4 Modul IP Client ditambahkan fitur Multi-threaded processing, selengkapnya klik disini.
