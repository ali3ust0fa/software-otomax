DigiPOS Terpadu

Sistem DigiPOS dapat diakses melalui Aplikasi Mobile, Web dan UMB (USSD Menu Browser) atau Dial dengan 2 pilihan Metode Pembayaran, yaitu LINKAJA dan NGRS. Disini akan dibahas tata cara mengintegrasikan DigiPOS dengan OtomaX menggunakan UMB dan metode pembayaran LINKAJA untuk mentransaksikan semua produk yang tersedia, seperti pulsa reguler, pulsa internet, paket SMS, paket nelpon dan lain sebagainya.

Berikut ini langkah mengintegrasikan sistem DigiPOS basis UMB atau Dial di OtomaX dengan pengaturan terpadu.

Kode Provider

Masuk menu Administrasi -> Provider -> buat Kode Provider.

Kode Produk External

Kode Produk External ialah Kode Produk yang nantinya dapat diakses oleh semua Reseller dan yang akan dimunculkan di Aplikasi Android (jika pakai). Buat seperti biasa.

Kode Produk Internal

Kode Produk Internal ialah Kode Produk yang nantinya hanya dapat diakses oleh 1 Reseller Internal. Berikut ini daftarnya.

Reseller Internal

Reseller Internal ialah Reseller milik server sendiri yang akun Jabber/XMPP-nya dipasang di IM Center.

  1. Buat akun Jabber/XMPP untuk Reseller Internal, klik disini.
  2. Klik Administrasi -> Grup Reseller buat Grup khusus untuk Reseller Internal, misal: RSI.

  3. Klik Administrasi -> Reseller -> pada Kode misal isi: RSI123 -> Nama isi: Reseller Internal -> Grup pilih RSI -> Limit Saldo isi: -100000000000 -> PIN isi misal: 8729fwef8743jew9347 -> klik tombol simpan.

  4. Klik Administrasi -> Pengirim -> di Tipe pilih Gtalk/XMPP -> Pengirim isi akun Jabber/XMPP Reseller Internal -> Kode Reseller isi: RSI123 -> klik tombol Simpan.

IM Hybrid HTH Internal

IM Hybrid HTH Internal ialah modul yang akan menembak ke IM Center milik server itu sendiri (bukan milik supplier) dan produk yang ditembak adalah Kode Produk Internal.

  1. Pilih IM Center -> klik kanan -> pilih Tambah IM Center... -> pilih tab: Jabber/XMPP -> di Label misal isi: #1 Reseller Internal -> di Username dan Password isi username dan password akun Jabber/XMPP Reseller Internal -> centang checkbox: Ignore offline messages -> di box: Filter Server isi tanda bintang (*) -> pilih tombol Save.

  2. Buat penangkap jawaban sesuai balasan milik server sendiri, klik disini. Misalnya kelompok jawabannya diberi nama MyServer.
  3. Pilih IM Hybrid HTH -> klik kanan -> pilih Tambah IM Hybrid HTH... -> pilih tab: Page 1 -> di Label misal isi: #1 MyServer -> Kelompok Jawaban pilih MyServer -> XMPP dari pilih akun Jabber/XMPP Reseller Internal -> centang checkbox: Hanya kirim jika YM/GTalk lawan online -> Parameter Parsing isi: pin=1234 (ganti 1234 dengan pin Reseller Internal) -> Kirim perintah ke isi akun Jabber/XMPP yang dijadikan Center di server sendiri -> klik tombol Simpan.

  4. Klik Administrasi -> klik Parsing ke Provider -> di Modul pilih #1 MyServer -> Produk isi Kode Produk External misal: S5 -> di kolom Perintah isi misal: IPR5.[tujuan].[pin]; terakhir klik tombol Simpan. Perhatikan gambar di bawah ini:

Alur Kerja

  1. Pengguna melakukan transaksi memakai Kode Produk External ke server.
  2. Reseller Internal menggunakan modul IM Hybrid HTH Internal akan memproses dengan menembakkannya ke IM Center, dan kemudian diproses oleh Dial DigiPOS.
  3. DigiPOS memberikan balasan dan ditangani oleh penangkap jawaban Dial DigiPOS.
  4. IM Center mengirim balasan ke IM Hybrid HTH Internal lalu ditangani oleh penangkap jawaban IM Hybrid HTH Internal.
  5. Server memberikan balasan status transaksi ke pengguna.
  6. Di menu Transaksi akan muncul 2 transaksi, transaksi dengan IM Hybrid HTH dan Dial DigiPOS.
  7. Selesai.
