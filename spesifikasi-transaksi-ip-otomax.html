Spesifikasi Transaksi IP OtomaX (3.6.8)

Tentu bila Anda sebagai supplier dan Mitra Anda ingin host to host via IP, mereka akan bertanya kepada Anda: bagaimana spesifikasi transaksi IP OtomaX?. Sebelum membahas itu Anda perlu mengetahui hal-hal yang harus dipenuhi untuk transaksi via IP, selengkapnya klik disini.

Spesifikasi IP OtomaX ialah sebagai berikut:

  1. Tujuan pengiriman transaksi
    http://IP:Port/. Contoh: http://123.456.789.10:212/

  2. Metode
    HTTPGET

  3. Syntax

    Uraian Format
    Cek saldo balance?memberID=[memberid]&sign=[sign]
    Transaksi trx?memberID=[memberid]&product=[product]&dest=[tujuan]&refID=[trxid]&sign=[sign]
    Cek status transaksi trx?memberID=[memberid]&product=[product]&dest=[tujuan]&refID=[trxid]&sign=[sign]&check=1
    Rumus Sign encodeBase64(sha1("OtomaX|" + memberId + "|" + product + "|" + dest + "|" + refID + "|" + pin + "|" + password))

    Adapun untuk contoh penerapan Rumus Sign (OtomaX) klik disini. Ini hanya ditujukan untuk developer software lain, sehingga bila Anda bukan developer tidak perlu memikirkan Rumus Sign tersebut.

    Parameter Keterangan
    [memberid] Kode Agen Reseller, contoh: AA001, MM0100
    [product] Kode Produk, contoh: S10, X5
    [tujuan] nomor tujuan transaksi, contoh: 08123456789
    [trxid] nomor urut transaksi, contoh: 100303, 2352627
    [sign] signature OtomaX, contoh: Cp9NLVuKWBt5Sey_uMlYsvnKvJI
    dest destination atau nomor tujuan transaksi, contoh: 081123456789
    refID nomor urut transaksi, contoh: 100303, 2352627
    pin pin untuk transaksi yang didapat saat pendaftaran Reseller, contoh: 80809933
    password password yang diberikan ketika mendaftar transaksi via IP, contoh: 3E#Ss100

Informasikan spesifikasi di atas kepada Mitra Anda agar mereka dapat melakukan setting di server mereka. Kemudian, bila Anda memiliki pertanyaan-pertanyaan seputar tehnis transaksi via IP di OtomaX silahkan kirimkan pertanyaan Anda ke alamat email support@otomax-software.com atau klik disini.

Catatan 1: Mulai versi 3.6.9 syntax transaksi IP bisa tanpa Sign (OtomaX). Info lengkap klik disini.

Catatan 2: Selain mendukung metode HTTP GET, mulai versi 3.7.0 juga mendukung metode HTTP POST. Info lengkap klik disini.
